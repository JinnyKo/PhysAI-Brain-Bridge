# PhysAI-Brain-Bridge
현시점 피지컬 AI와 로보틱스의 결합에서 발생하는 **'지능의 병목'**을 해결하기 위한 온디바이스 인지 엔진 & 통신 브리지 프로젝트입니다. ===> 많은 테스트들을 해볼 예정
단순히 시키는 동작만 하는 로봇이 아니라, 인간의 말을 이해하고 스스로 업무를 "해-나가는", (굳이 하나로 정한다기 보다) 마치 인터스텔라TARS 같은 지능형 로봇 구축을 목표로 합니다.

---

##  프로젝트 개요

지금까지의 개발 경험 >> 로봇 하드웨어 제조사가 열어준 제한적인 SDK,API 안에서만 놀거나, 네트워크 레이턴시 때문에 사람 같은 반응과 액션에 한계가 있었음. 
저는 이 문제를 해결하기 위해 **맥미니(Brain)**와 **로봇(Body)**을 고속으로 물려, 음성 기반 의도 분류의 추론이 시간으로 동작하는 아키텍처를 우선으로 테스트할 예정입니다. 

---

##  아키텍처 (Current Phase)

일단 단계별로 테스트를 하기 위해 다음과 같이 레이어를 구성했습니다.

### 1. Cognitive Layer (맥미니 M4 Pro)

* **Brain:** 음성 인식(Whisper/V2V) 및 의도 분류 모델 구동.
* **Reasoning:** "인사해", "설거지해" 같은 명령을 받고 로봇이 무엇을 할지 최종 결정.

### 2. Bridge Layer (Initial: 맥미니 통합 / Future: NX 분리)

* **Wrapper:** 맥미니의 추론 결과물(Intent)을 로봇 제조사의 숨겨진 함수(SDK)로 매핑.
* **Communication:** 일단은 맥미니에서 로봇으로 직접 쏘는 구조로 가되, 추후 NX 내부로 이 레이어를 분리해서 최적화할 예정.

### 3. Physical Layer (Robot NX/SBC)

* **Control:** 제조사의 Closed ROS 2 기반 로우레벨 제어.
* **Action:** 실제 모터 구동 및 센서 데이터 피드백.

---

## 우선순위 테스트 항목 (Focus Items)

지금 이 레포에서 제가 가장 집중해서 테스트할 부분들입니다.

* **[ ] 유선 직결 성능 체킹:** Wi-Fi(AP/WLAN) 모드의 레이턴시 한계를 극복하기 위해 유선 LAN으로 물렸을 때의 데이터 무손실 및 0-레이턴시 구현.
* **[ ] 의도 분류 아키텍처:** 단순 명령 수행을 넘어, 로봇의 센서 상태와 비전 데이터를 통합해 '진짜 의도'를 파악하는 멀티모달 추론.
* **[ ] 오디오 가속 파이프라인:** 모터 소음이 심한 환경에서도 정확하게 음성을 따내서 맥미니로 쏴주는 고성능 오디오 인터페이스 연동.
* **[ ] SDK 함수 맵핑:** 제조사가 NX 안에 숨겨둔 함수들을 밖에서 얼마나 빠르고 정확하게 호출(Invoke)할 수 있는지 브리지 효율성 테스트.

---

## 해결해야 할 과제 (Known Issues)

* **Closed Architecture:** 제조사 SDK가 캡슐화되어 있어 모터의 Raw 데이터를 직접 건드리기 어려움 -> 고성능 브리지 코드로 극복 예정.
* **Computing Power:** OpenVLA 등 거대 모델을 온디바이스에서 실시간으로 돌리기 위한 최적화 작업 필요.
* **Synchronization:** 맥미니의 판단 속도와 로봇의 물리적 동작 속도 사이의 동기화(Sync) 문제.
